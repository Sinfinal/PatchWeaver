# 百炼应用 ID 使用手册

本文档以应用 ID `e27a6aabb75d4760a9b62e63f9124643` 为例，说明如何通过阿里云百炼 DashScope 的 OpenAI 兼容接口调用应用。

## 1. ID

**百炼应用 ID**

调用时需要把它放进接口地址里，形成专属应用的请求入口。

## 2. 基本信息

- 应用 ID：`e27a6aabb75d4760a9b62e63f9124643`
- 接口模式：OpenAI Compatible
- 请求方法：`POST`
- 接口路径：`/api/v2/apps/agent/{APP_ID}/compatible-mode/v1/responses`

实际地址：

```text
https://dashscope.aliyuncs.com/api/v2/apps/agent/e27a6aabb75d4760a9b62e63f9124643/compatible-mode/v1/responses
```

## 3. 前置条件

- 已开通百炼服务
- 已获取百炼 API Key
- 已创建并发布对应应用
- 本地已安装 `curl` 或 OpenAI SDK

## 4. 请求头

```http
Content-Type: application/json
Authorization: Bearer <YOUR_API_KEY>
```

## 5. 最小可用调用

### curl

```bash
curl -X POST "https://dashscope.aliyuncs.com/api/v2/apps/agent/e27a6aabb75d4760a9b62e63f9124643/compatible-mode/v1/responses" \
  -H "Authorization: Bearer <YOUR_API_KEY>" \
  -H "Content-Type: application/json" \
  -d '{
    "input": "你好"
  }'
```

### Python

```python
from openai import OpenAI
import os

client = OpenAI(
    api_key=os.getenv("DASHSCOPE_API_KEY"),
    base_url="https://dashscope.aliyuncs.com/api/v2/apps/agent/e27a6aabb75d4760a9b62e63f9124643/compatible-mode/v1"
)

resp = client.responses.create(
    input="你好"
)

print(resp.model_dump_json(indent=2))
```

## 6. 流式调用

如果要边生成边返回，给请求体加上 `stream: true`。

```bash
curl -X POST "https://dashscope.aliyuncs.com/api/v2/apps/agent/e27a6aabb75d4760a9b62e63f9124643/compatible-mode/v1/responses" \
  -H "Authorization: Bearer <YOUR_API_KEY>" \
  -H "Content-Type: application/json" \
  -d '{
    "input": "请简要介绍你自己",
    "stream": true
  }'
```

## 7. 多轮对话

把历史消息按数组传入 `input`。

```bash
curl -X POST "https://dashscope.aliyuncs.com/api/v2/apps/agent/e27a6aabb75d4760a9b62e63f9124643/compatible-mode/v1/responses" \
  -H "Authorization: Bearer <YOUR_API_KEY>" \
  -H "Content-Type: application/json" \
  -d '{
    "input": [
      {
        "type": "message",
        "role": "user",
        "content": [
          { "type": "input_text", "text": "你是谁？" }
        ]
      },
      {
        "type": "message",
        "role": "assistant",
        "content": [
          { "type": "input_text", "text": "我是一个 AI 助手。" }
        ]
      },
      {
        "type": "message",
        "role": "user",
        "content": [
          { "type": "input_text", "text": "你能做什么？" }
        ]
      }
    ]
  }'
```

## 8. 常见问题

- `401/403`：API Key 无效、过期或权限不足
- `404`：应用 ID 不存在，或 URL 拼写错误
- 没有流式返回：请求体未设置 `stream: true`
- 工作流应用不流式：需要在应用中开启流式输出并重新发布

## 9. 建议用法

- API Key 放环境变量，不要硬编码
- 先用 `curl` 验证，再接入代码
- 生产环境固定使用同一个应用 ID
